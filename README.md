# Customer Churn Prediction

This repository contains a churn analysis and classification workflow for telecom customers. It is designed to illustrate how to explore customer behavior, prepare churn data, and build a predictive classification model.

## Repository contents
- `Customer Churn Analysis and Prediction.ipynb` — primary notebook with the full analysis pipeline
- `telecom_churn.csv` — dataset used for exploration, feature engineering, and model training
- `.gitignore` — repository ignore rules

## What’s included
- data overview and exploratory analysis
- feature preparation and data cleaning
- model training for churn prediction
- evaluation of classification performance
- interpretation of results and business insights

## Purpose
This project is intended as a learning example for classification problems, with a focus on:
- understanding churn drivers in telecom data
- comparing model performance
- building a reproducible data science workflow

## How to use
Open `Customer Churn Analysis and Prediction.ipynb` in Jupyter or VS Code and run the notebook cells sequentially. The notebook is the main entry point and contains both code and analysis narrative.

## Notes
- The notebook is the recommended way to explore the data and reproduce the workflow.
- The dataset is stored locally in `telecom_churn.csv` and is used directly by the notebook.
